FOOTBALL ANALYTICS PLATFORM V1 - CLEAN SETUP

PART 1 - RESET SUPABASE AND CREATE TABLES
1. Open Supabase.
2. Open your project.
3. Click SQL Editor.
4. Click inside the big SQL box.
5. Press Ctrl+A, then Delete.
6. Open this file in the ZIP: supabase/schema.sql
7. Copy everything in schema.sql.
8. Paste it into Supabase SQL Editor.
9. Click Run.

This schema deletes the old prototype tables first, then creates the clean V1 tables.

PART 2 - GET SUPABASE KEYS
1. In Supabase, click Project Settings.
2. Click API.
3. Copy Project URL.
4. Copy anon public key.

PART 3 - RUN LOCALLY FIRST
1. Install Node.js from nodejs.org if you do not have it.
2. Open this folder in VS Code or Terminal.
3. Copy .env.example and rename the copy to .env
4. Put your Supabase URL and anon key into .env
5. Run: npm install
6. Run: npm run dev
7. Open the local website shown in the terminal.

PART 4 - USE IT
1. Click Import PFF CSV/XLSX.
2. Choose your PFF file.
3. Name the project.
4. Chart on desktop, iPad, or iPhone.
5. Export CSV when done.

FORMATION RULES IN V1
T + 31 = T31
TT + 31 = TT31
B before formation = B21
B after formation = 21B
21 + WS = 21 WS
21 + WW = 21 WW
21 + DT = 21 DT
21 + DW = 21 DW

FIELD ZONE LOGIC
Manual fields:
- FZ Advantage
- FZ Attacked

Automatic field on export/database:
- Advantage Attacked = 1 when FZ Advantage equals FZ Attacked
- Advantage Attacked = 0 when they do not match
